# Product

## Register

product

## Users

Developers, students, IT professionals, and homelab enthusiasts in Indonesia and beyond. They arrive to evaluate 8Labs (marketing pages) or to get unblocked while running a lab (docs, guides, troubleshooting). Docs readers are mid-task: terminal open, copying commands, often on a second monitor or a phone.

## Product Purpose

8labs.id is the single public site for 8Labs: marketing home, product docs (`/docs`), hands-on guides (`/guides`), and blog (`/blog`). Docs were merged in from docs.8labs.id; success means one consistent brand surface where old deep links still resolve and content is easy to search and read.

## Brand Personality

Practical, techy, friendly. "Tukang ngoprek" energy: hands-on, no fluff, real commands that work. Confident but not corporate.

## Anti-references

- Generic SaaS template look (the site started as Astroship; the brand has outgrown it)
- Default Docusaurus/Starlight theme left unthemed — docs must read as 8Labs, not as the framework
- Heavy enterprise docs portals (IBM-style): slow, dense chrome, weak typography

## References

- Cognivis / meinGPT / Klea: light enterprise-AI blueprint, isometric line-art, technical grid
- 8labs.id home: light blueprint system — white paper, neutral zinc ink, 8Labs red accent, sharp radius (no pills), hairline grid, crosshair registration marks, mono micro-labels, isometric SVG motifs. See DESIGN.md.

## Design Principles

- Brand continuity first: docs, blog, and home are one site (same fonts, same red accent, same sharp blueprint language — no pills)
- Content is the interface: typography and code blocks carry the experience; chrome stays quiet
- Every legacy URL keeps working: redirects are part of the product
- Fast on cheap hardware: static output, no client framework on content pages

## Accessibility & Inclusion

WCAG 2.1 AA target. Body text ≥4.5:1 contrast in both themes. Dark mode is a first-class theme, not an inversion afterthought. Respect prefers-reduced-motion.
